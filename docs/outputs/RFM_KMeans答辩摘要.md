# RFM + KMeans 答辩摘要

## 一、模型选择结果

本次实验围绕特征筛选与聚类效果提升进行了两轮优化。

第一轮优化中，我对 `k=3、4、5、6` 四组聚类数进行了对比，在精简比例特征集 `compact_ratio` 下，结果如下：

| K 值 | 轮廓系数 |
|---|---:|
| 3 | 0.2700 |
| 4 | 0.1938 |
| 5 | 0.2239 |
| 6 | 0.1640 |

第二轮优化中，我进一步做了特征消融实验，重点测试：

- 是否保留 `engagement_score`
- 是否保留 `member_level_score`
- 是否同时移除两者

最终比较结果如下：

| 特征方案 | 轮廓系数 |
|---|---:|
| compact_no_engagement_no_member | 0.3542 |
| compact_no_engagement | 0.3305 |
| compact_no_member | 0.2907 |
| compact_ratio | 0.2700 |
| quality_no_browse | 0.2117 |

最终最优方案为：

- **最佳 K 值：3**
- **最佳轮廓系数：0.3542**
- **最佳特征方案：compact_no_engagement_no_member**

结论：

- 相比最初 `k=4` 时的 `0.1999`，通过特征消融和比例特征压缩，聚类效果提升明显；
- 说明 `engagement_score` 与 `member_level_score` 在当前数据中存在一定冗余，移除后反而提高了群体区分度；
- 虽然 `0.3542` 仍未达到 `0.4`，但已经接近课程设计中较好的聚类效果水平。

## 二、聚类群体命名

根据 `k=3` 的聚类结果，将三个群体命名为：

| Cluster | 业务命名 | 典型特征 |
|---|---|---|
| Cluster 0 | 高价值核心群体 | 订单频次高、总消费金额高、客单价高 |
| Cluster 1 | 流失风险群体 | 最近消费间隔长、订单少、消费金额低 |
| Cluster 2 | 潜力成长群体 | 消费中等，但收藏率、加购率、综合行为得分较高 |

---

## 三、RFM 与 KMeans 对照结果

| Cluster | 一般用户 | 流失风险用户 | 潜力用户 | 高价值用户 |
|---|---:|---:|---:|---:|
| 0 | 1021 | 559 | 1042 | 548 |
| 1 | 45 | 534 | 0 | 0 |
| 2 | 401 | 237 | 394 | 219 |

关键结论：

1. **Cluster 0 聚集了最多高价值用户**
   - 高价值用户数达到 `650`
   - 说明 KMeans 可以较稳定识别高价值用户群体

2. **Cluster 1 聚集了最多流失风险用户**
   - 流失风险用户数达到 `690`
   - 说明 KMeans 对低活跃、低消费用户也具有较强识别能力

3. **Cluster 2 更适合作为潜力用户池**
   - 收藏率、加购率、行为活跃度较高
   - 具备后续运营激活价值

## 四、适合答辩直接讲的结论

可以直接表述为：

> 在用户聚类部分，我使用 Spark MLlib 的 KMeans 算法，对用户的消费行为、订单特征和画像特征进行了分群分析。实验先比较了不同 `k` 值，再进一步围绕 `engagement_score` 和 `member_level_score` 做特征消融，最终发现去掉这两个冗余特征后，聚类效果提升最明显。最终最优方案选取 `k=3`，轮廓系数达到 `0.3542`，相比初始结果提升显著。结合聚类特征均值和 RFM 对照结果可以发现，Cluster 0 明显对应高价值核心群体，Cluster 1 明显对应流失风险群体，Cluster 2 则体现出较强的潜力成长特征。这说明 KMeans 聚类结果与 RFM 规则分层在核心用户识别上具有较好的一致性，也证明了机器学习方法可以有效补充规则分层分析。*
